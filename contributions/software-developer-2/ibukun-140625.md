Here’s a clear, developer‑ready proposal your factory agent can act on. It combines feel, layout, and implementation hooks so it can reshape the site coherently.

🎯 Site Direction: “Reassuring, real, and motivating – not corporate”
This microsite should feel like:

“Someone slightly older than me explaining uni honestly, not a brochure trying to sell it.”

Tone:

Honest > polished
Supportive > pushy
Clear > academic


🎨 1. Updated Colour System (lively + trustworthy)
CSS:root {  --primary: #2563EB;      /* Trust + clarity (main actions) */  --accent: #FF7A18;       /* Energy + motivation (CTAs, highlights) */  --positive: #22C55E;     /* Growth + reassurance (success states) */  --background: #FFFDF8;   /* Warm neutral (less sterile than white) */  --text: #1F2937;         /* Strong readable body text */}``Show more lines
Usage rules:

Blue = structure + trust
Orange = action only
Green = reassurance (“you’re on the right track”)
Warm background = avoids “school portal” vibe


🧱 2. Layout Structure (post-hero)
Section flow (critical)
✅ 1. “Is uni right for me?” (decision framing)
NOT “About us”

3–4 large cards:

[ I want to know what uni is like ]
[ I'm unsure if it's worth it ]
[ I want to explore options ]
[ I feel pressured to go ]

👉 This reframes the site around user mindset, not content

✅ 2. Real student stories (trust engine)
Grid or horizontal scroll:
[Photo]  “I thought uni wasn’t for me…”
[Photo]  “Best decision I made…”
[Photo]  “I chose NOT to go…”

👉 Include non-uni paths to increase credibility

✅ 3. “What uni actually involves” (demystify)
Break into short cards:

Teaching style
Social life
Costs
Freedom vs structure

👉 Avoid long paragraphs — chunk everything

✅ 4. Path comparison (VERY important)
University   | Apprenticeship | Gap Year
-------------|----------------|---------
Cost         | Earn money     | Flexible
Independence | Structured     | Risk/uncertain

👉 Sixth-formers are really comparing options

✅ 5. Primary CTA section (sticky reinforcement)
“Still unsure? Let’s figure it out together”
Buttons:

Talk to someone
Explore options

Staggered cards (delay: 50–100ms)

📱 5. Mobile-first tweaks (non-negotiable)

1-column layout everywhere
Cards = full width
Buttons ≥ 48px height
Max text width ~65ch equivalent

👉 Assume user is on phone, slightly distracted

🧠 6. UX Psychology (important)
Avoid:

“Apply now”
“Join us”
Overly polished messaging

Use instead:

“Explore whether uni is right for you”
“See what it’s actually like”
“No pressure — just information”

👉 Reduces resistance and builds trust
