You are improving the accessibility of a single‑page charity website using CSS only (no HTML or JS changes unless unavoidable).
Apply or recommend clear CSS styling indicators to support accessibility best practices, focusing on:

Readable text (comfortable font size, line height, spacing)
Sufficient colour contrast for text, buttons, and links
Clear visual hierarchy for headings
Obvious interactive elements (links and buttons look clickable)
Visible keyboard focus states for all interactive elements

Constraints:

Keep changes minimal and fast to implement
Assume the site will be built or adjusted in about 1 hour
Prefer simple, widely supported CSS
Avoid advanced or experimental features

Output:

Provide example CSS rules that demonstrate these indicators
Use sensible defaults (e.g. :focus-visible, relative units, safe colours)
Do not include explanations longer than a single short comment per section

Do not:

Add animations that could affect motion sensitivity
Remove outlines without replacing them
Make compliance or legal claims
