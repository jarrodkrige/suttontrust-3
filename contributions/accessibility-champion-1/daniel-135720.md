Review and modify the following existing single‑page charity website to improve accessibility.
Important constraints

Do not invent, rewrite, or add new site content
Do not make up image meanings, alt text context, headlines, copy, labels, or section purposes
Only work with the structure and content that already exists
If information needed for accessibility is missing, use neutral placeholders (e.g. “Descriptive image”) or leave a comment explaining what is required

The site includes:

A hero section with headline, text, and a primary CTA
One or two content sections
Images
A simple contact or donation form
Basic navigation or anchor links

Goals

Make high‑impact accessibility improvements that are realistic for a ~1‑hour build
Improve usability for keyboard users, screen‑reader users, and people with low vision or cognitive differences
Align loosely with WCAG 2.2 Level A and essential AA (no compliance claims)

What to prioritise

Semantic HTML structure (headings, landmarks, navigation)
Readable text and sufficient colour contrast
Accurate, non‑speculative alt text (or placeholders when intent is unclear)
Clearly associated form labels and accessible errors
Logical focus order and visible keyboard focus states

How to apply changes

Improve markup, attributes, and styles without changing meaning
Preserve visual design unless a change is required for accessibility
Prefer simple, readable solutions
Avoid refactoring, frameworks, or feature additions

When information is missing

Do not guess
Add a brief comment noting what a human editor must supply (e.g. meaningful alt text, error message copy)

Required output

Updated markup / styles / behaviour with accessibility improvements applied
Short inline comments where accessibility depends on unknown content

Success criteria

Page is fully usable with keyboard only
Content remains unchanged in meaning and wording
Accessibility improvements are practical and minimal
